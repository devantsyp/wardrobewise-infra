<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Configure AWS S3 bucket and add environment variables to Render</name>
  <what-built>S3 storage backend is configured in prod.py (Plan 02-01). The code is ready — but the actual AWS bucket and credentials don't exist yet.</what-built>
  <instructions>
    **Step A: Create an S3 bucket**
    1. Go to AWS Console -> S3 -> Create bucket
    2. Bucket name: choose something like `laundryadvisor-media` (must be globally unique)
    3. Region: `us-east-1` (or your preferred region — note the region name)
    4. **Uncheck "Block all public access"** — confirm the warning
    5. Leave all other settings as defaults
    6. Click "Create bucket"

    **Step B: Add bucket policy for public read**
    1. Go to the newly created bucket -> Permissions tab -> Bucket policy
    2. Paste this policy (replace `YOUR-BUCKET-NAME` with your actual bucket name):
    ```json
    {
        "Version": "2012-10-17",
        "Statement": [
            {
                "Sid": "PublicReadGetObject",
                "Effect": "Allow",
                "Principal": "*",
                "Action": "s3:GetObject",
                "Resource": "arn:aws:s3:::YOUR-BUCKET-NAME/*"
            }
        ]
    }
    ```
    3. Save

    **Step C: Create IAM credentials**
    1. Go to AWS Console -> IAM -> Users -> Create user
    2. User name: `laundryadvisor-s3`
    3. Attach policy: `AmazonS3FullAccess` (or a custom policy scoped to your bucket)
    4. Create the user -> Security credentials -> Create access key
    5. Choose "Application running outside AWS"
    6. Copy the Access Key ID and Secret Access Key (you won't see the secret again)

    **Step D: Add env vars to Render**
    1. Go to Render Dashboard -> your web service -> Environment
    2. Add these 4 environment variables:
       - `AWS_STORAGE_BUCKET_NAME` = your bucket name (e.g., `laundryadvisor-media`)
       - `AWS_S3_REGION_NAME` = your region (e.g., `us-east-1`)
       - `AWS_ACCESS_KEY_ID` = from Step C
       - `AWS_SECRET_ACCESS_KEY` = from Step C
    3. Save — Render will redeploy automatically

    **Step E: Also add to your local `.env` file** (for local S3 testing if desired):
    ```
    AWS_STORAGE_BUCKET_NAME=laundryadvisor-media
    AWS_S3_REGION_NAME=us-east-1
    AWS_ACCESS_KEY_ID=your-key
    AWS_SECRET_ACCESS_KEY=your-secret
    ```
    (Note: in dev mode, local filesystem is used by default. S3 is only used in prod settings.)
  </instructions>
  <resume-signal>Type "done" when all 4 env vars are configured in Render and the bucket is created, or describe any issues.</resume-signal>
