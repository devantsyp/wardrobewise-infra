  <action>
    **1. Add dependencies to requirements.txt:**
    - `django-storages[s3]>=1.14,<2.0`
    - `Pillow>=10.0,<12.0`

    **2. Create `wardrobe/` app directory** with `__init__.py`, `apps.py`.

    In `wardrobe/apps.py`:
    - Standard AppConfig with `name = 'wardrobe'`
    - Override `ready()` to import `wardrobe.signals` (registers the post_delete handler)

    **3. Create `wardrobe/models.py`:**

    Two `upload_to` callables at module level:
    - `garment_photo_path(instance, filename)` -> `f"user_{instance.user.id}/garment_{instance.pk}/garment.jpg"`
    - `care_label_path(instance, filename)` -> `f"user_{instance.user.id}/garment_{instance.pk}/care_label.jpg"`

    IMPORTANT: These callables require `instance.pk` to be set. The create view (Plan 02-02) MUST use a two-step save: save the record first (without files), then assign file fields and save again.

    `Garment` model:
    - `user` = ForeignKey to `settings.AUTH_USER_MODEL`, `on_delete=CASCADE`, `related_name='garments'`
    - `name` = CharField(max_length=100)
    - `category` = CharField(max_length=50, choices=CATEGORY_CHOICES)
    - `color` = CharField(max_length=100, blank=True)
    - `fabric` = CharField(max_length=100, blank=True)
    - `notes` = TextField(max_length=500, blank=True)
    - `garment_photo` = ImageField(upload_to=garment_photo_path, blank=True, null=True)
    - `care_label_photo` = ImageField(upload_to=care_label_path, blank=True, null=True)
    - `created_at` = DateTimeField(auto_now_add=True)
    - `updated_at` = DateTimeField(auto_now=True)
    - `class Meta: ordering = ['-created_at']`
    - `def __str__(self): return self.name`

    CATEGORY_CHOICES (15 categories, per user decision — this list is LOCKED):
    ```python
    CATEGORY_CHOICES = [
        ("T-Shirts & Tops", "T-Shirts & Tops"),
        ("Shirts", "Shirts"),
        ("Jeans", "Jeans"),
        ("Pants", "Pants"),
        ("Dresses", "Dresses"),
        ("Skirts", "Skirts"),
        ("Jackets & Blazers", "Jackets & Blazers"),
        ("Coats & Outerwear", "Coats & Outerwear"),
        ("Shorts", "Shorts"),
        ("Sweaters & Knitwear", "Sweaters & Knitwear"),
        ("Hoodies & Sweatshirts", "Hoodies & Sweatshirts"),
        ("Underwear & Loungewear", "Underwear & Loungewear"),
        ("Socks", "Socks"),
        ("Activewear", "Activewear"),
        ("Sleepwear & Robes", "Sleepwear & Robes"),
    ]
    ```

    **4. Create `wardrobe/signals.py`:**
    - `@receiver(post_delete, sender=Garment)` handler `delete_garment_files`
    - If `instance.garment_photo`: call `instance.garment_photo.delete(save=False)`
    - If `instance.care_label_photo`: call `instance.care_label_photo.delete(save=False)`

    **5. Update settings:**

    `base.py`:
    - Add `'wardrobe'` to INSTALLED_APPS (after `'accounts'`)

    `dev.py`:
    - Add `MEDIA_URL = '/media/'`
    - Add `MEDIA_ROOT = BASE_DIR / 'media'`

    `prod.py`:
    - EXTEND (do NOT replace) the existing STORAGES dict. The `'staticfiles'` key already exists for WhiteNoise. Add the `'default'` key:
      ```python
      STORAGES["default"] = {
          "BACKEND": "storages.backends.s3.S3Storage",
          "OPTIONS": {
              "bucket_name": env("AWS_STORAGE_BUCKET_NAME"),
              "region_name": env("AWS_S3_REGION_NAME", default="us-east-1"),
              "access_key": env("AWS_ACCESS_KEY_ID"),
              "secret_key": env("AWS_SECRET_ACCESS_KEY"),
              "querystring_auth": False,
              "file_overwrite": True,
              "default_acl": None,
          },
      }
      ```
    - CRITICAL: Use `STORAGES["default"] = {...}` NOT `STORAGES = {...}` — the latter would wipe the WhiteNoise staticfiles entry.

    **6. Run migrations:**
    - `pip install -r requirements.txt` (installs django-storages and Pillow)
    - `python manage.py makemigrations wardrobe`
    - `python manage.py migrate`

    **7. Add `media/` to .gitignore** if not already present (dev uploads go here).

    **1. Create `wardrobe/forms.py`:**

    `GarmentForm` (ModelForm for Garment):
    - `class Meta: model = Garment`
    - `fields = ['name', 'category', 'color', 'fabric', 'notes', 'garment_photo', 'care_label_photo']`
    - Exclude `user` (set in view), `created_at`, `updated_at`

    Widget attrs for Tailwind styling (match Phase 1 patterns from accounts/forms.py):
    - All text inputs: `class: 'w-full px-4 py-3 rounded-lg border border-gray-300 focus:ring-2 focus:ring-dark-teal-500 focus:border-transparent transition'`
    - `name`: placeholder "e.g., Blue Oxford Shirt"
    - `category`: use Select widget with same border/focus classes
    - `color`: placeholder "e.g., Navy blue"
    - `fabric`: placeholder "e.g., 100% cotton"
    - `notes`: Textarea with `rows=3`, placeholder "Any special care notes..."
    - File inputs: `class: 'hidden'`, `accept: 'image/jpeg,image/png'` (we'll use a custom label for styling)

    Custom validation — override `clean_garment_photo()` and `clean_care_label_photo()`:
    - If file is provided:
      - Check content_type is in `['image/jpeg', 'image/png']` — raise ValidationError "Only JPG and PNG files are accepted."
      - Check file.size <= 10 * 1024 * 1024 (10 MB) — raise ValidationError "File size must not exceed 10 MB."
    - Return the cleaned file

    Extract the validation into a helper `_validate_image(self, file, field_name)` to avoid duplication.

    **2. Create `wardrobe/admin.py`:**

    `GarmentAdmin(admin.ModelAdmin)`:
    - `list_display = ('name', 'category', 'user', 'created_at')`
    - `list_filter = ('category', 'created_at')`
    - `search_fields = ('name', 'color', 'fabric', 'notes')`
    - `readonly_fields = ('created_at', 'updated_at')`

    Register: `admin.site.register(Garment, GarmentAdmin)`
  </action>
