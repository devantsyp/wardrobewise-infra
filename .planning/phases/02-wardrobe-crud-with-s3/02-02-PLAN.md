  <action>
    **1. Create `wardrobe/views.py` with 5 function-based views:**

    All views use `@login_required` decorator. All single-garment views use:
    `garment = get_object_or_404(Garment, pk=pk, user=request.user)` — returns 404 (not 403) for other users' garments, per user decision.

    **`garment_list(request)`:**
    - Query: `Garment.objects.filter(user=request.user)` (ordering is default from model Meta: `-created_at`)
    - Render `wardrobe/wardrobe_list.html` with `{'garments': garments}`

    **`garment_detail(request, pk)`:**
    - Get garment with user isolation
    - Render `wardrobe/garment_detail.html` with `{'garment': garment}`

    **`garment_create(request)`:**
    - GET: render empty GarmentForm
    - POST: validate form. If valid:
      1. `garment = form.save(commit=False)`
      2. `garment.user = request.user`
      3. `garment.save()` — pk now assigned
      4. If `'garment_photo' in request.FILES`: `garment.garment_photo = request.FILES['garment_photo']`
      5. If `'care_label_photo' in request.FILES`: `garment.care_label_photo = request.FILES['care_label_photo']`
      6. `garment.save()` — second save uploads files to S3 with pk in path
      7. Redirect to `wardrobe:garment_detail` with `pk=garment.pk`
    - CRITICAL: Two-step save required because `upload_to` callables use `instance.pk`, which is None on first save.
    - Pass `{'form': form, 'action': 'create'}` to template

    **`garment_edit(request, pk)`:**
    - Get garment with user isolation
    - GET: render GarmentForm with `instance=garment`
    - POST: validate form with `request.FILES` and `instance=garment`. If valid:
      1. If `'garment_photo' in request.FILES` and `garment.garment_photo`: `garment.garment_photo.delete(save=False)` — delete old file before uploading new one
      2. If `'care_label_photo' in request.FILES` and `garment.care_label_photo`: `garment.care_label_photo.delete(save=False)`
      3. `form.save()` — saves all fields including new files
      4. Redirect to `wardrobe:garment_detail`
    - Pass `{'form': form, 'garment': garment, 'action': 'edit'}` to template

    **`garment_delete(request, pk)`:**
    - Require POST method only (GET returns 405 or redirect)
    - Get garment with user isolation
    - `garment.delete()` — the post_delete signal (from 02-01) handles S3 file cleanup
    - Redirect to `wardrobe:garment_list`

    **2. Create `wardrobe/urls.py`:**

    ```python
    urlpatterns = [
        path('', views.garment_list, name='garment_list'),
        path('<int:pk>/', views.garment_detail, name='garment_detail'),
        path('add/', views.garment_create, name='garment_create'),
        path('<int:pk>/edit/', views.garment_edit, name='garment_edit'),
        path('<int:pk>/delete/', views.garment_delete, name='garment_delete'),
    ]
    ```

    - Keep `path('', include('core.urls'))` but the `/wardrobe/` route in core/urls.py must be REMOVED (see step 4)
    - In dev only, add media URL serving. Add at the bottom:
      ```python

      from django.conf.urls.static import static
      if settings.DEBUG:
          urlpatterns += static(settings.MEDIA_URL, document_root=settings.MEDIA_ROOT)
      ```
    - NOTE: `settings.MEDIA_URL` and `settings.MEDIA_ROOT` are only defined in dev.py. In prod, S3 serves files directly. The `if settings.DEBUG` guard + `hasattr` or try/except handles this gracefully. Actually, `static()` returns empty list if MEDIA_URL is not set, so just check DEBUG.

    - IMPORTANT: `LOGIN_REDIRECT_URL = '/wardrobe/'` in base.py still works because the wardrobe app now serves that URL.

    - `MEDIA_ROOT = BASE_DIR / 'media'`
