**Constants:**
```python
INPUT_COST_PER_TOKEN = Decimal('0.0000025')    # $2.50 / 1M input tokens (GPT-4o)
OUTPUT_COST_PER_TOKEN = Decimal('0.00001')     # $10.00 / 1M output tokens (GPT-4o)
DAILY_LIMIT = 10
BUDGET_GUARD_USD = Decimal('9.00')
```

**Exceptions:**
```python
class RateLimitExceeded(Exception): pass
class BudgetGuardTripped(Exception): pass
class AnalysisError(Exception): pass
```

**Module-level client:**
```python
client = OpenAI()  # reads OPENAI_API_KEY from environment
```

**Helper functions:**

`_daily_count(user) -> int`: Query `UsageLog.objects.filter(user=user, created_at__date=timezone.now().date()).count()`.

`_budget_exceeded() -> bool`: Query `UsageLog.objects.aggregate(t=Sum('cost_usd'))['t'] or Decimal('0')`, return `total >= BUDGET_GUARD_USD`.

`_image_hash(image_bytes: bytes) -> str`: Return `hashlib.sha256(image_bytes).hexdigest()`.

`budget_guard_active() -> bool`: Public helper for views/context — same logic as `_budget_exceeded()`. Needed so views can check budget status without calling analyze.

`get_daily_count(user) -> int`: Public helper wrapping `_daily_count()` for context processor use.

**`_call_api(image_bytes, fabric_hint="") -> tuple[dict, object]`:**
- Base64 encode image_bytes
- System prompt: "You are a laundry care expert. Analyze the care label in the image. Return a JSON object with exactly these keys: summary, washing, drying, ironing, bleach, is_delicate. For any field you cannot determine from the label, use the string 'Unable to determine'. is_delicate must be a boolean (true if the item requires special/delicate handling, false otherwise). Return clean, direct instructions only — no caveats about image quality."
- User message: "Analyze this care label." + optional " Fabric: {fabric_hint}." if provided
- model="gpt-4o"
- response_format={"type": "json_object"}
- max_tokens=500
- Parse response.choices[0].message.content with json.loads()
- Return (parsed_dict, response.usage)

**`analyze_care_label(garment, user) -> CareAnalysis`:**
1. Check budget guard — raise `BudgetGuardTripped()` if exceeded
2. Check rate limit — raise `RateLimitExceeded()` if `_daily_count(user) >= DAILY_LIMIT`
3. Read image bytes ONCE: `garment.care_label_photo.open('rb'); image_bytes = garment.care_label_photo.read(); garment.care_label_photo.close()` — CRITICAL: FieldFile is a stream, read once only
4. Compute image hash
5. Dedup check: `CareAnalysis.objects.filter(image_hash=img_hash).first()`
   - If cached exists AND cached.garment == garment: set `from_cache=True`, save, return it
   - If cached exists AND cached.garment != garment: extract AI fields from cached record, set from_cache=True, skip API call
   - If no cache: call `_call_api(image_bytes, fabric_hint=garment.fabric)`, set from_cache=False
6. In `transaction.atomic()`:
   - Delete existing CareAnalysis for this garment (handles re-analyze case)
   - Create new CareAnalysis with all ai_* fields and matching user-editable fields (copy ai values to user fields initially)
   - If not from_cache (real API call): create UsageLog with `cost_usd = Decimal(usage.prompt_tokens) * INPUT_COST_PER_TOKEN + Decimal(usage.completion_tokens) * OUTPUT_COST_PER_TOKEN`
7. Return the new CareAnalysis

**Error handling:** Wrap the API call in try/except. On any openai exception, raise `AnalysisError(str(e)) from e`.

**Important implementation notes (from research):**
- Use `garment.care_label_photo.open('rb')` before `.read()` to ensure the file is open, then `.close()` after — handles both local and S3 storage backends
- Never use float for cost — always Decimal
- The `raw_ai_json` field stores the parsed dict (not the raw string) — JSONField handles serialization
- For the dedup cross-garment case, store `raw_ai_json=cached.raw_ai_json` — copy the full original JSON

```python
from wardrobe.services.analysis import get_daily_count, budget_guard_active, DAILY_LIMIT


def daily_usage_counter(request):
    """Inject daily usage count into template context.

    Only runs the DB query on wardrobe namespace pages to avoid
    unnecessary queries on admin, auth, and other pages.
    """
    if not request.user.is_authenticated:
        return {}

    # Only inject on wardrobe namespace pages
    try:
        if request.resolver_match is None or request.resolver_match.app_name != 'wardrobe':
            return {}
    except AttributeError:
        return {}

    count = get_daily_count(request.user)
    return {
        'daily_usage_count': count,
        'daily_usage_limit': DAILY_LIMIT,
        'budget_guard_tripped': budget_guard_active(),
    }
```

2. **Update `laundry_advisor/settings/base.py`:** Add `'wardrobe.context_processors.daily_usage_counter'` to the `context_processors` list inside `TEMPLATES[0]['OPTIONS']`, after the existing `'django.contrib.messages.context_processors.messages'` line.

Note: The context processor also injects `budget_guard_tripped` so templates can show budget guard messages without extra view logic.
