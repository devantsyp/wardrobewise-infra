## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding (`px-1`, `gap-1`) |
| sm | 8px | Compact element spacing, badge margins (`px-2`, `gap-2`) |
| md | 16px | Default element spacing, card padding (`p-4`, `px-6`) |
| lg | 24px | Section padding, between header and grid (`mb-6`, `gap-6`) |
| xl | 32px | Page section breaks (`py-8`, `mb-8`) |
| 2xl | 48px | Major page-level vertical rhythm (`py-12`) |
| 3xl | 64px | Empty-state vertical centering (`py-16` equivalent) |

Exceptions:
- Sticky bottom bar: fixed bottom-0, full viewport width, `py-4 px-6` inner padding
- Garment card thumbnails in load cards: 64px square (`w-16 h-16`)
- Touch target minimum on interactive cards: 44px height — achieved via `py-3` on buttons

## Component Inventory

All components follow established patterns from phases 1-3.

### Garment Selection Card (basket page)
- Base: `rounded-lg shadow-sm overflow-hidden bg-white` (identical to wardrobe grid card)
- Selected state: add `ring-2 ring-deep-space-400 border-2 border-deep-space-400`
- Checkbox: `absolute top-2 left-2`, 20px square, `accent-deep-space-500` or custom SVG checkmark
- Unselectable (no analysis): hidden from DOM entirely per BSKT-02 decision
- Card info: aspect-square photo + garment name (`text-sm font-semibold text-deep-space-800 px-2 pt-2 truncate`) + category (`text-xs text-thistle-500 px-2 pb-2`)
- No click navigation — clicking card toggles checkbox only

### Controls Row (above grid)
- Layout: `flex items-center gap-4 mb-4`
- Left cluster: [Category `<select>`] [Select All `<button>`] [Clear All `<button>`]
- Right cluster: `ml-auto text-sm text-thistle-500` — "X selected / Y available"
- Category select: `border border-deep-space-200 rounded-md px-3 py-2 text-sm bg-white text-deep-space-800`
- Select All / Clear All: `text-sm font-semibold text-deep-space-600 hover:text-deep-space-900 underline` (text buttons, no border)
- Inline notice for 21st garment: amber toast inline below controls row — `text-sm text-amber-700 bg-amber-50 border border-amber-200 rounded-md px-3 py-2`

### Sticky Bottom Bar
- Position: `fixed bottom-0 inset-x-0 z-40`
- Background: `bg-white border-t border-deep-space-100 shadow-lg`
- Inner: `max-w-7xl mx-auto px-6 py-4 flex items-center justify-between`
- Mobile: `flex-col gap-3` with button going full-width (`w-full`)
- Button (enabled): `bg-dark-teal-600 hover:bg-dark-teal-700 text-white font-semibold px-6 py-3 rounded-lg transition-colors`
- Button (disabled, <2 selected): `bg-deep-space-200 text-deep-space-400 font-semibold px-6 py-3 rounded-lg cursor-not-allowed` with `title` attribute tooltip
- Main content bottom padding: `pb-24` to prevent content hidden behind bar

### Loading Skeleton (plan computing state)
- Three placeholder cards, each `rounded-lg bg-deep-space-100 animate-pulse h-32 mb-4`
- Shown immediately when user triggers plan generation
- Replaced by actual load cards once `group_into_loads()` returns

### Load Card
- Base: `bg-white rounded-lg shadow-sm overflow-hidden mb-4`
- Header: `bg-deep-space-700 text-white px-4 py-3 font-semibold text-sm`
  - Text: "Load N: [Color Group] — [Temp] ([X garments])"
- Body: `px-4 py-4`
  - Thumbnail row: `flex flex-wrap gap-3 mb-4`
  - Each thumbnail: `w-16 h-16 rounded-md object-cover` + garment name `text-xs text-deep-space-700 text-center mt-1 w-16 truncate`
  - Warning badge on thumbnail: `absolute top-0 right-0 bg-amber-400 text-amber-900 text-xs font-semibold px-1 rounded`
  - Grouping rationale: `text-xs text-thistle-500 mb-3` — "Grouped by: Color (Darks) | Temp (30°C)"
  - Wash settings: `text-sm text-deep-space-800` per setting line
  - Summary warnings: `mt-3 pt-3 border-t border-deep-space-100` — `flex flex-wrap gap-2`, each warning as `text-xs font-semibold bg-amber-100 text-amber-800 px-2 py-1 rounded-full`

### Special Care Card
- Identical structure to Load Card except:
  - Header: `bg-amber-500 text-white` (amber instead of deep-space-700)
  - Header text: "Special Care ([X garments])"
  - Body: flat list `flex flex-col gap-3`, each row `flex items-center gap-3`
  - Row: `w-12 h-12 rounded-md object-cover` + garment name `text-sm font-semibold text-deep-space-800` + care method `text-xs text-thistle-500`

### Basket Selector Dropdown (near page heading)
- Layout: `flex items-center gap-3 mb-2`
- Label: `text-sm font-semibold text-thistle-500`
- Select: `border border-deep-space-200 rounded-md px-3 py-2 text-sm bg-white text-deep-space-800 font-semibold`
- "+ New Basket" trigger: `text-sm font-semibold text-dark-teal-600 hover:text-dark-teal-700 underline ml-2`
- "Manage Baskets" link: `text-sm text-thistle-500 hover:text-deep-space-700 underline ml-2`

### Modal (new basket + manage baskets + delete confirmation)
- Overlay: `fixed inset-0 bg-black/50 z-50 flex items-center justify-center`
- Panel: `bg-white rounded-xl shadow-xl max-w-md w-full mx-4 p-6`
- Title: `text-xl font-semibold text-deep-space-900 mb-4`
- Input (name field): `w-full border border-deep-space-200 rounded-lg px-3 py-2 text-sm text-deep-space-800 focus:outline-none focus:ring-2 focus:ring-deep-space-400`
- Actions: `flex justify-end gap-3 mt-6`
- Cancel/Discard: `text-sm font-semibold text-thistle-500 hover:text-deep-space-700 px-4 py-2`
- Confirm (create/rename): `bg-dark-teal-600 hover:bg-dark-teal-700 text-white font-semibold px-4 py-2 rounded-lg text-sm`
- Delete confirm: `bg-red-600 hover:bg-red-700 text-white font-semibold px-4 py-2 rounded-lg text-sm`

## Interaction Contract

These behaviors are locked by CONTEXT.md and must be implemented as specified.

| Interaction | Behaviour |
|-------------|-----------|
| Click garment card | Toggle checkbox selection; no navigation |
| Click Select All | Select all analyzed garments regardless of active category filter |
| Click Clear All | Deselect all garments regardless of active category filter |
| Select 21st garment | Show inline "Maximum 20 garments reached" notice; ignore the selection |
| Category filter change | Filter visible cards; does not change existing selections |
| Click "Create Laundry Plan (X)" | Smooth scroll to plan results section; show loading skeleton; render plan |
| Selection changes after first plan generated | Auto-update plan (live); show skeleton during recalculation |
| Click "Save Plan" | POST to save endpoint; show success toast; update saved plan in DB |
| Return to basket with saved plan | Show plan immediately on page load |
| Click "+ New Basket" | Open new basket modal |
| Submit new basket form | Create basket; add to dropdown; set as active |
| Click basket in dropdown | Switch active basket; load that basket's selections and saved plan |
| Click "Manage Baskets" | Open manage baskets modal/page |
| Click Rename in manage view | Inline edit or sub-modal with current name pre-filled |
| Click Delete in manage view | Show confirmation modal with basket name; require explicit confirm |
| Confirm delete | Delete basket; if active basket deleted, switch to first remaining or show empty state |

## Accessibility Contract

- Garment card checkboxes: `<input type="checkbox">` associated with garment name via `aria-label` or `<label for>`
- Sticky button disabled state: use `aria-disabled="true"` and `disabled` attribute; tooltip via `title` attribute
- Modal: `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to modal title; focus trapped within modal while open; ESC closes modal
- Loading skeleton: `aria-busy="true"` on plan results section during computation; `aria-live="polite"` on plan container for screen reader updates
- Category filter: native `<select>` element for full keyboard support
- Load cards: use `<h3>` for card headers to preserve heading hierarchy (h1 page → h2 section → h3 card)
- Warning badges: include `aria-label` describing the warning for screen readers
