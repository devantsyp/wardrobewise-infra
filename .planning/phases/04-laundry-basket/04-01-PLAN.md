    6. Create `laundry/services/grouping.py` with `group_into_loads(garments: list[dict]) -> dict`:

       The function takes a list of dicts with keys: pk, name, photo_url, category, color, washing, drying, bleach, is_delicate.

       **Step 1 - Special care routing:**
       Check `washing` field (lowercased) for keywords: 'dry clean', 'dryclean', 'dry-clean', 'hand wash only', 'hand-wash only', 'hand wash', 'do not wash', 'do not machine wash', 'spot clean', 'no washing'. Route matches to `special_care` list with `care_method` derived from matched keyword (e.g., "Dry clean only", "Hand wash only", "Do not wash", "Spot clean only").

       **Step 2 - Color group classification:**
       Lowercase `color` field. Check against ordered keyword lists:
       - DARKS: 'black', 'navy', 'dark blue', 'dark grey', 'dark gray', 'charcoal', 'dark brown', 'dark green', 'dark red', 'dark'
       - WHITES: 'white', 'cream', 'ivory', 'off-white', 'off white'
       - Default: 'lights'
       Check darks first (longest keywords first), then whites, default lights.

       **Step 3 - Temperature extraction:**
       Parse from `washing` field:
       1. Regex `r'(\d+)\s*(?:°[Cc]|degrees?\s*[Cc]|[Cc]\b)'` with re.IGNORECASE for explicit Celsius
       2. Keyword fallback: 'cold'->30, 'cool'->30, 'warm'->40, 'hot'->60
       3. None if unparseable

       **Temperature bucket normalization:** Round to nearest of [30, 40, 60]. <=35->30, 36-50->40, >50->60.

       For display: temp_label = f"{bucket}°C" if bucket is not None from original parse. If original parse returned None, temp_label = "Coolest wash" and bucket = 30.

       **Step 4 - Grouping key:** `(color_group, temperature_bucket, is_delicate_cycle)`
       Where `is_delicate_cycle` matters only when both delicate and non-delicate garments exist in the same (color_group, temp_bucket). If all garments in a (color_group, temp_bucket) are delicate, they form one load with cycle='delicate'. If mixed, split into two loads.

       Implementation: First group by (color_group, temp_bucket). Then within each group, check if mixed. If mixed, split into delicate and normal sub-loads. If all same, single load.

       **Step 5 - Warning extraction per garment:**
       Scan `drying`, `bleach`, `washing` fields (all lowercased):
       - 'air dry', 'lay flat', 'line dry', 'hang to dry' -> "air dry only"
       - 'no bleach', 'do not bleach', 'non-chlorine', 'bleach free' -> "no bleach"
       - 'delicate cycle', 'gentle cycle', 'delicate wash' -> "delicate cycle"
       - 'do not tumble dry', 'no tumble' -> "no tumble dry"
       Each warning added at most once per garment.

       **Step 6 - Build load dicts:**
       Each load: `{'color_group': str, 'temperature': int, 'temp_label': str, 'cycle': 'normal'|'delicate', 'garments': [{'pk': int, 'name': str, 'photo_url': str|None, 'warnings': [str]}], 'warnings': [str]}`.
       Load-level `warnings` = deduplicated union of all garment warnings in the load.

       **Step 7 - Sort loads descending by len(garments).**

       Return `{'loads': [...], 'special_care': [...]}`.
