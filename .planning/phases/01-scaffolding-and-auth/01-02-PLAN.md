       - Override password field widget attrs similarly
       - Do NOT override `clean()` or error handling — the view handles error display

    2. Create `accounts/views.py` with three views:

       **register_view** (POST + GET):
       - GET: render empty RegistrationForm
       - POST: validate form. On success:
         - `user = form.save()`
         - `login(request, user)` — auto-login after registration
         - `messages.success(request, "Welcome to Wardrobe Wise!")` — flash message
         - Redirect to `'/wardrobe/'` (will 404 until Phase 2 — that's expected)
       - POST on failure: re-render form with errors (standard bound form behavior)

       **login_view** (POST + GET):
       - GET: render empty LoginForm; pass `next` from GET params to template context
       - POST: create `LoginForm(request, data=request.POST)`. On success:
         - `login(request, form.get_user())`
         - `messages.success(request, "You're logged in!")` — flash message
         - Redirect to `request.POST.get('next') or '/wardrobe/'`
       - POST on failure — per user decision, clear ALL fields and show generic error:
         - Create a FRESH `LoginForm()` (not the bound form — this clears all fields)
         - Add a non-field error: the fresh form should carry the error "Invalid email or password." — store this in context as a separate variable (e.g., `login_error = "Invalid email or password."`) since a fresh unbound form won't carry form errors
         - Render with the fresh form + login_error in context
         - Pass `next` value back to template context to preserve redirect target

       **logout_view:**
       - Call `logout(request)`
       - `messages.info(request, "You've been logged out.")` — flash message
       - Redirect to `'/login/'`

    3. Create `accounts/urls.py`:
       ```python

       from . import views

       urlpatterns = [
           path('register/', views.register_view, name='register'),
       ```
